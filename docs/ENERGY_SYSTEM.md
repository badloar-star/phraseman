# Система Энергии - Документация

## Обзор

Система энергии используется для ограничения количества уроков, которые пользователь может пройти в день. Каждый урок требует 1 единицу энергии для начала.

## Механика

### Базовые параметры
- **Максимальная энергия**: 5 единиц
- **Расход на урок**: 1 единица при начале урока (только начало, не за каждый ответ)
- **Восстановление**: +1 единица каждые 2 часа (автоматически)
- **Хранилище**: AsyncStorage с ключом `energy_state`

### Состояние энергии
```typescript
interface EnergyState {
  current: number;        // Текущая энергия (0-5)
  max: number;           // Максимальная энергия (5)
  lastRecoveryTime: number; // Timestamp последнего восстановления
}
```

## API

### `getEnergyState(): Promise<EnergyState>`
Получить текущее состояние энергии из хранилища.
- Если данные отсутствуют, инициализирует с максимальной энергией.
- Возвращает `EnergyState`.

### `checkAndRecover(): Promise<EnergyState>`
Проверить и применить восстановление энергии.
- Восстанавливает +1 энергию каждые 2 часа до максимума.
- Вызывайте эту функцию перед началом урока.
- Возвращает обновленное состояние.

### `spendEnergy(amount?: number): Promise<boolean>`
Потратить энергию.
- По умолчанию тратит 1 единицу.
- Возвращает `true` если энергия была потрачена, `false` если энергии недостаточно.

### `addEnergy(amount?: number): Promise<EnergyState>`
Восстановить энергию вручную.
- По умолчанию добавляет 1 единицу.
- Не превышает максимум.
- Используется для премиум-бонусов или достижений.

### `resetEnergyToMax(): Promise<EnergyState>`
Сбросить энергию на максимум.
- Используется для тестирования или специальных ситуаций.

### `getTimeUntilNextRecovery(): Promise<number>`
Получить время до следующего восстановления в миллисекундах.
- Возвращает 0 если энергия уже на максимуме.

### `formatTimeUntilRecovery(ms: number): string`
Форматировать время восстановления в читаемый формат.
- Пример: "1ч 30м" или "30м"

## Интеграция в приложение

### 1. Проверка энергии перед началом урока (lesson1.tsx)

```typescript
import { spendEnergy, checkAndRecover } from './energy_system';

// В loadData():
const energyState = await checkAndRecover();
if (energyState.current <= 0) {
  setShowEnergyModal(true);
  return; // Не загружаем урок
}

// После успешной загрузки урока:
await spendEnergy(1);
```

### 2. UI отображение энергии (home.tsx)

Компонент энергии отображается на главном экране:
- Прогресс-бар с текущим количеством энергии
- Статус восстановления (время до следующей энергии)
- Обновляется при каждом фокусе на таб Home

```typescript
import { getEnergyState, getTimeUntilNextRecovery, formatTimeUntilRecovery } from './energy_system';

// В loadData():
const [energyState, nextRecoveryTime] = await Promise.all([
  getEnergyState(),
  getTimeUntilNextRecovery(),
]);

setEnergyCurrent(energyState.current);
setEnergyMax(energyState.max);
setTimeUntilRecovery(nextRecoveryTime);
```

### 3. Модальное окно недостаточной энергии

При попытке начать урок без энергии показывается модальное окно с сообщением.

## Тестирование

Для сброса энергии в разработке:
```typescript
import { resetEnergyToMax } from './app/energy_system';

// Сбросить энергию на максимум
await resetEnergyToMax();
```

## Примечания

1. **Энергия тратится только при начале урока**, не за каждый ответ
2. **Ошибки в уроке НЕ влияют на энергию** — сбрасывается только стрик, но энергия остается потраченной
3. **Восстановление происходит автоматически** — система проверяет при загрузке главного экрана
4. **Максимум 5 уроков в день** — пользователь может пройти максимум 5 уроков, затем должен ждать восстановления энергии
