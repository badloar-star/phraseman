# План интеграции системы энергии

## Быстрый старт

### 1. Основной модуль
Файл: `/app/energy_system.ts`

Функции:
- `getEnergyState()` - получить текущую энергию
- `checkAndRecover()` - проверить и восстановить энергию
- `spendEnergy()` - потратить энергию за урок
- `addEnergy()` - добавить энергию (для премиум)
- `resetEnergyToMax()` - сброс для тестирования

### 2. Проверка энергии в уроках
Файл: `/app/lesson1.tsx`

Что было добавлено:
```typescript
import { spendEnergy, checkAndRecover } from './energy_system';

// В loadData():
const energyState = await checkAndRecover();
if (energyState.current <= 0) {
  setShowEnergyModal(true);
  return; // Не загружаем урок
}

// После загрузки урока:
await spendEnergy(1);
```

Модальное окно показывается при недостатке энергии.

2. **Проверка восстановления энергии:**
   - В консоли браузера разработчика (DevTools) выполните:
   ```javascript

   const newState = {
     current: 0,
     max: 5,
     lastRecoveryTime: Date.now() - 2.5 * 60 * 60 * 1000
   };
   AsyncStorage.setItem('energy_state', JSON.stringify(newState));
   ```
   - Обновите страницу
   - Энергия должна восстановиться до 1

3. **Сброс энергии для разработки:**
   ```javascript
   import { resetEnergyToMax } from './app/energy_system';
   await resetEnergyToMax();
   ```

## Соответствие требованиям

- ✓ Макс энергия: 5 единиц
- ✓ Восстановление: +1 каждые 2 часа
- ✓ Урок тратит: -1 энергию (только при начале)
- ✓ Ошибка в уроке: -0 энергии (стрик сбрасывается отдельно)
- ✓ Хранилище: AsyncStorage 'energy_state'
- ✓ Энергия спендится только при начале урока, не за ответы

## Вопросы и ответы

**В: Почему энергия восстанавливается за 2 часа, а не за 1?**
А: По требованиям спеки. Это делает систему менее агрессивной к пользователю.

**В: Может ли пользователь получить энергию за достижения?**
А: Да, функция `addEnergy()` позволяет добавить энергию. Это можно вызвать из системы достижений.

**В: Что происходит при ошибке?**
А: Ошибка НЕ влияет на энергию (она уже потрачена). Сбрасывается только стрик.

**В: Как энергия хранится?**
А: В AsyncStorage под ключом `energy_state` с полной информацией о времени последнего восстановления.
