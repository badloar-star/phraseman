# PhraseMan — Руководство по контент-аудиту

## Как запускать аудит

Запусти 5 агентов параллельно (каждый читает свой файл вручную, без скриптов):

```
Agent 1: lesson_data_1_8.ts
Agent 2: lesson_data_9_16.ts
Agent 3: lesson_data_17_24.ts
Agent 4: lesson_data_25_32.ts
Agent 5: lesson_cards_data.ts + constants/lessons.ts
```

Используй промпт из файла `docs/AUDIT_PROMPT.md`.

## Что проверяется

### 1. Переводы на русский
- Точные, а не примерные ("obey" ≠ "уважать")
- Естественные для носителя русского языка
- Порядок слов естественный ("сегодня утром", не "этим утром")
- Нет добавленных слов которых нет в оригинале
- Нет пропущенных слов из оригинала
- Управление падежами ("заслуживает отдыха", не "отдых")

### 2. Английские фразы
- Грамматически верные (артикли, числа существительных, предлоги)
- `furniture`, `money`, `chess`, `news` — неисчислямые, нельзя `those`
- `for + период` (for three hours), не `during + период`
- Артикли перед существительными: `signs a contract`, `find a solution`
- Идиоматичность: `all day yesterday` не `all yesterday day`

### 3. Дистракторы
- Нет дубликатов внутри одного массива `distractors: []`
- Все слова — реальные английские слова (не `'agoo'`, `'hersy'`, `'bookedy'`)
- Нет правильных альтернатив среди дистракторов (`gray` не дистрактор к `grey`)
- Нет собственных имён как дистракторов к нарицательным

### 4. Предлоги (глобальный английский)
- Ориентир — глобальный английский, не только британский
- `on/in the street` — оба варианта правильны → нужны `alternatives`
- `on/in the menu` — `on` распространённее глобально
- `at/on the weekend` — оба правильны (уже обработано в уроке 8)
- Если возможны несколько правильных предлогов — пометить все в `alternatives`

### 5. Соответствие words[] и english:
- Слова в массиве `words` должны точно совпадать с английской фразой
- Частая ошибка: слово в `words` одно, в фразе другое (peeled vs cleaned)

### 6. Карточки (lesson_cards_data.ts)
- Факты в `secretRu` — проверить на достоверность (нет мифов)
- Нет технического мусора в полях (заголовки, вопросы-артефакты ИИ)
- Нет дублирования одного факта в разных карточках
- Терминология: "наречие частоты", не "маркер времени"
- Нет смешения языков в украинских полях (русские слова в `uk` полях)

### 7. Педагогика
- Уровень сложности соответствует номеру урока (1-8 = A0, 9-16 = A2, 17-24 = B1, 25-32 = B2)
- Каждый урок = одна тема (не 3 темы одновременно)
- Фразовый глагол в уроке фразовых глаголов — главная тема, не грамматика наречий
- Сложные слова (desperate, seldom) с пометкой о регистре

### 8. Украинские переводы (uk)
- Согласование рода/числа (та гостра сокира, не той гострий сокиру)
- Нет русских слов в украинских полях
- Соответствует смыслу (не "вона голодна" для "she has an appetite")

## Аудио — ОБЯЗАТЕЛЬНО после исправлений

**При любом изменении поля `english:`:**
```bash
OPENAI_API_KEY=$OPENAI_API_KEY node scripts/regen_phrase_audio.mjs "Исправленная фраза"
```
Обновить ключ в `assets/audio/en/index.json`.

## Частые паттерны ошибок (следить при генерации новых уроков)

1. ИИ добавляет слова в перевод которых нет в оригинале
2. ИИ путает `during` и `for` для периодов времени
3. ИИ забывает артикли в английских фразах
4. ИИ дублирует слова в дистракторах
5. ИИ создаёт псевдослова с суффиксами -y/-ly/-edy
6. ИИ вставляет технические артефакты (заголовки) в поля данных
7. ИИ путает `those/these` и `there/here` в переводах
8. ИИ переводит `then` как "сейчас/сегодня" вместо "тогда"
9. ИИ не согласует род/число в украинских переводах
