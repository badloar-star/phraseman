# PhraseMan — Чеклист производительности для тестеров

**Целевые устройства:** слабые Android (Motorola moto g 5G, Redmi, Samsung A-series)  
**Как запустить Metro логи:** `npx expo start` → смотреть терминал во время тестирования

---

## Подготовка

1. Включить **Performance Monitor** на устройстве:
   - Потрясти телефон (shake) → выбрать **"Perf Monitor"**
   - Смотреть на `JS: RAM` и `UI: FPS` во время переходов
   - Норма: JS FPS ≥ 50, RAM < 200 MB
   - Проблема: JS FPS < 30 или RAM > 300 MB

2. Очистить кэш приложения перед тестом (Settings → Apps → PhraseMan → Clear Cache)

3. Зарядить телефон до 100% (throttling режим Battery Saver искажает результаты)

## Сценарий 1 — Навигация между экранами (главная жалоба)

**Цель:** воспроизвести зависание при переходах

| Шаг | Действие | Ожидаемый результат | Фактический результат |
|-----|----------|--------------------|-----------------------|
| 1 | Открыть приложение | Главный экран за < 3 сек | |
| 2 | Нажать на урок (напр. Урок 1) | Переход без зависания, < 1 сек | |
| 3 | Пройти 2-3 вопроса | Кнопки реагируют сразу | |
| 4 | Нажать "Назад" | Плавный возврат на главную | |
| 5 | Повторить шаги 2-4 ещё 3 раза | Каждый переход < 1 сек | |
| 6 | Перейти на вкладку Квизы | Плавный переход | |
| 7 | Вернуться на Главную | Плавный переход | |
| 8 | Нажать Словарь | Плавный переход | |

**Что смотреть в логах Metro:**
```
[PERF] nav → lesson_menu: XXms
[PERF] home:loadData: XXms
```
Норма: < 200ms. Проблема: > 500ms.

## Сценарий 2 — Нагрузка через 5 минут использования

**Цель:** проверить утечки памяти и накопленный lag

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Пройти 3 урока подряд | FPS не падает от урока к уроку |
| 2 | Открыть Достижения | Экран открывается < 1 сек |
| 3 | Открыть Карточки | Экран открывается < 1 сек |
| 4 | Вернуться на Главную | FPS ≥ 50 в Perf Monitor |
| 5 | Проверить RAM | < 250 MB |

```
- Шаг X завис на: [сколько секунд]
- JS FPS во время зависания: [число из Perf Monitor]
- RAM во время зависания: [число из Perf Monitor]

- RAM после 3 уроков: [число]

[вставить сюда]
```

## Как включить детальные логи (для разработчика)

В приложении перейти: Настройки → О приложении → нажать 7 раз на версию → появится **"Dev Mode"**

В Dev Mode включить: **"Performance Logging"** — в Metro терминале появятся строки `[PERF]`.
