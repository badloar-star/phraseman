# Руководство Тестирования Системы Блокировки Уроков

## Быстрая Проверка

### 1. Проверка UI (Визуальное)

**Шаг 1: Запустить приложение**
```bash
npm start
```

**Шаг 2: Открыть экран меню урока (Урок 2)**
- Тап на "УРОК 2" в главном меню
- Или используй `lesson_menu.tsx` с `id=2`

**Ожидаемый результат:**
- Кнопка "Начать урок" имеет иконку 🔐 (замок)
- Кнопка выглядит disabled (полупрозрачная, opacity: 0.5)
- Цвет фона кнопки серый (`bgMuted`)

**Шаг 3: Клик на заблокированный урок**
- Тап на кнопку "Начать урок"

**Ожидаемый результат:**
- Появляется модальное окно (modal)
- Текст: "🔐 Урок заблокирован"
- Сообщение: "Сначала пройди Урок 1 с оценкой ≥ 4.5"
- Кнопка "Понимаю"

### 2. Проверка Разблокировки (На Эмуляторе)

**Шаг 1: Пройти Урок 1**
1. Открыть Урок 1 (всегда доступен)
2. Ответить правильно на все 50 вопросов
3. Получить оценку 5.0 (все правильно)
4. Экран `lesson_complete` должен показать медаль

**Шаг 2: Проверить разблокировку**
1. Вернуться на экран меню
2. Открыть Урок 2
3. **Ожидание:** Иконка замка исчезла, кнопка активна (полная видимость)
4. Нажать "Начать урок" - должен открыться нормальный экран урока

**Шаг 3: Проверить что Урок 3 всё ещё заблокирован**
1. Открыть Урок 3
2. **Ожидание:** Иконка замка, модальное окно "Пройди Урок 2 с оценкой ≥ 4.5"

### 3. Проверка Минимальной Оценки

**Сценарий: Недостаточная оценка**
1. Открыть Урок 1
2. Ответить на 49 вопросов правильно (оценка 4.9)
3. Вернуться на экран меню
4. **Ожидание:** Урок 2 всё ещё заблокирован 🔐
5. Сообщение: "Нужно минимум 4.5"

**Сценарий: Ровно 4.5**
1. Открыть Урок 1
2. Ответить на ровно 45 вопросов правильно (4.5 = 45/50 * 5)
3. Вернуться на экран меню
4. **Ожидание:** Урок 2 разблокирован ✅

## Отладка

### Проверить Статус Уроков в Консоли

Добавить в любой компонент:
```typescript
import { getUnlockedLessonIds, getLessonLockInfo } from './lesson_lock_system';

useEffect(() => {
  (async () => {
    const unlockedIds = await getUnlockedLessonIds();
    console.log('Разблокированные уроки:', unlockedIds);

    // Проверить конкретный урок
    const info = await getLessonLockInfo(5);
    console.log('Урок 5 информация:', info);
  })();
}, []);
```

### Вывод в Консоли:
```
Разблокированные уроки: [1, 2, 5, 6]
Урок 5 информация: {
  isLocked: false,
  lockedByLesson: null,
  minRequiredScore: 0
}
```

```typescript
import { resetLockSystem } from './lesson_lock_system';

// Сбросить все разблокировки (оставить только Урок 1)
await resetLockSystem();
```

## Критерии Успеха

### ✅ Всё работает правильно, если:

1. **Урок 1 доступен**
   - Нет иконки замка 🔐
   - Кнопка полностью видна (opacity: 1)
   - Можно открыть урок с первого клика

2. **Урок 2 заблокирован по умолчанию**
   - Иконка замка 🔐 видна
   - Кнопка полупрозрачная (opacity: 0.5)
   - При клике показывается модальное окно

3. **Модальное окно содержит:**
   - Большую иконку замка (56px)
   - Заголовок "Урок заблокирован"
   - Сообщение о требуемом уроке и оценке
   - Кнопка "Понимаю" для закрытия

4. **Разблокировка работает:**
   - Урок 1 с оценкой ≥ 4.5 → Урок 2 разблокирован
   - Это видно сразу после прохождения
   - Состояние сохраняется при перезагрузке

5. **Последовательная разблокировка:**
   - Урок 2 требует Урок 1 с score ≥ 4.5
   - Урок 3 требует Урок 2 с score ≥ 4.5
   - И так для всех уроков 2-32

### ❌ Ошибки для проверки:

- [ ] Урок 1 НЕ доступен (не должно быть)
- [ ] Урок 2 доступен без прохождения Урока 1
- [ ] Модальное окно не появляется
- [ ] Разблокировка не происходит после прохождения
- [ ] Иконка замка видна у открытого урока
- [ ] Оценка 4.4 разблокирует следующий урок (не должна)

## Дополнительные Проверки

### Проверка Языков

Если язык установлен на **Украинский (uk)**:
- Текст модального окна: "Урок заблокований" (не "заблокирован")
- Сообщение: "Спочатку пройди Урок 1..." (не "Сначала")
- Кнопка: "Розумію" (не "Понимаю")

### Проверка на Разных Устройствах

Протестировать на:
- [ ] iPhone (iOS)
- [ ] Android телефон
- [ ] Планшет
- [ ] Веб-версия (если есть)

Модальное окно должно корректно отображаться на всех размерах экрана.

### Проверка Performance

После разблокировки нескольких уроков:
- Приложение не должно замедляться
- Переходы между уроками должны быть гладкими
- AsyncStorage операции должны быть быстрыми (<100ms)

## Смешанные Сценарии

### Сценарий 1: Пользователь вернулся через неделю
1. Пользователь прошёл уроки 1-5 неделю назад
2. Закрыл приложение
3. Открыл приложение через неделю
4. **Ожидание:** Уроки 1-5 всё ещё разблокированы
5. Может продолжить с Урока 6 (заблокирован до прохождения Урока 5)

### Сценарий 2: Повторное прохождение
1. Пользователь разблокировал Урок 5
2. Вернулся на Урок 1 для практики
3. Прошёл Урок 1 ещё раз (для улучшения оценки)
4. **Ожидание:** Никаких изменений в статусе разблокировки
5. Уроки 2-5 остаются разблокированными как были

### Сценарий 3: Прогресс после каждого урока
1. Запустить приложение (только Урок 1 доступен)
2. Пройти Урок 1 с score 5.0 → Урок 2 разблокирован
3. Пройти Урок 2 с score 4.5 → Урок 3 разблокирован
4. Пройти Урок 3 с score 4.9 → Урок 4 разблокирован
5. И так далее...
6. **Ожидание:** Последовательное открытие доступа к урокам
